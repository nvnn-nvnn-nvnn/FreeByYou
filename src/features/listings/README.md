# Listings feature

Everything that powers the **listings grid** on the Home screen lives in this
folder. It renders a header, a scrollable category filter, and a 2-column grid
of items that loads more as you scroll (infinite scroll). It currently runs on
**placeholder data** — see "Swapping in real data" below.

```
┌─────────────────────────────────┐
│  Header  (⚙️   Title   🔍)        │   <- src/components/headerPlaceholder.tsx
├─────────────────────────────────┤
│  [ All ][ Food ][ Books ][ ... ] │   <- category-chips.tsx
├─────────────────────────────────┤
│  ┌─────────┐   ┌─────────┐       │
│  │ Item A  │   │ Item B  │       │   <- listing-card.tsx (x2 per row)
│  └─────────┘   └─────────┘       │
│        ... scrolls forever ...   │   <- FlatList in src/app/index.tsx
└─────────────────────────────────┘
```

## Files in this folder

| File | What it does |
|------|--------------|
| `listings.data.ts` | **The data layer.** Defines the `Listing` type, the `CATEGORIES` list, generates 240 placeholder listings, and exposes `fetchListings({ category, page })`. This is the ONLY file you change to use a real backend. |
| `categorize.ts` | **The auto-tagger.** Classifies a raw listing (title + description) into one genre via keyword matching. The seam for smarter ML/LLM tagging later. |
| `use-listings.ts` | **The pagination brain.** A hook that calls `fetchListings`, holds the list state, and exposes `loadMore`, `refresh`, `loading`, `refreshing`, and `reachedEnd`. Resets automatically when the category changes. |
| `category-chips.tsx` | The horizontal, scrollable row of category filter pills. Selected pill uses the theme's `tint` color. |
| `listing-card.tsx` | One grid cell: placeholder color thumbnail + title + neighborhood/distance. |
| `index.ts` | Barrel file — re-exports the public pieces so screens can `import { ... } from '@/features/listings'`. |
| `README.md` | This file. |

> The screen that assembles these lives at `src/app/index.tsx` (Expo Router
> requires route files to live under `src/app`, so it can't move into this
> folder). The header is the shared component at
> `src/components/headerPlaceholder.tsx`.

## How the data flows

```
src/app/index.tsx (HomeScreen)
   │  selected category (useState)
   ▼
useListings(category)              <- use-listings.ts
   │  calls on mount + when category changes, and on loadMore/refresh
   ▼
fetchListings({ category, page })  <- listings.data.ts  (PLACEHOLDER)
   │  returns { items, nextPage }
   ▼
FlatList renders items as <ListingCard /> in 2 columns
   onEndReached -> loadMore() -> next page appended
```

## Key behaviors

- **Infinite scroll:** `FlatList`'s `onEndReached` calls `loadMore`, which
  appends the next page. A spinner shows in the footer while fetching; a
  "You've reached the end." message shows when there are no more pages.
- **Category filter:** tapping a chip changes the `category` state. `useListings`
  watches `category` and resets to page 0 of that category.
- **Pull to refresh:** swipe down to re-fetch page 0 (`RefreshControl`).
- **Loading / empty states:** a centered spinner on first load and category
  switches; an empty message only after a finished load returns nothing.

## The category chips ARE the navigation

There is no separate tab navigator. The app is a single screen and the category
chips double as the nav bar:
`Explore · Furniture · Technology · Clothing · Food · Books · Toys · Tools · Other`.

- `Explore` is the generic "show everything" view (the app's landing feed) — NOT
  a genre an item is tagged with.
- `Other` is the fallback bucket for items the auto-tagger can't classify.

The layout is just: header (global, in `src/app/_layout.tsx`) → category chips →
listings grid.

## Auto-tagging pipeline (`categorize.ts`)

Real listings scraped from Facebook/Nextdoor/etc. arrive as free text with no
genre. `categorize({ title, description })` assigns each one to a single genre so
it appears under the right chip:

```
raw scraped item            categorize()              tagged listing
{ title: "Old laptop",  ->  keyword "laptop"/    ->   category: "technology"
  description: "..." }       "computer" matches        (shows under Technology)
```

Today it's a keyword matcher (the genre with the most keyword hits wins; no match
→ `other`). The placeholder data is itself generated by running raw samples
through `categorize()`, so what you see in the grid is the *real* tagging path.

To upgrade tagging later, replace the body of `categorize()` with an ML model or
an LLM call that returns an `ItemCategoryId` — callers don't change.

## Placeholder data

`listings.data.ts` builds 240 fake listings deterministically — realistic
titles per category (e.g. "Oak coffee table", "Cordless drill"), a neighborhood,
a distance, a "posted at" time, and a pastel thumbnail color (no images yet).
`fetchListings` adds a 600ms delay to mimic the network and serves them in
pages of 10.

## Swapping in real data

When the backend is ready, **only `listings.data.ts` changes**. Keep the
`fetchListings` signature and return shape identical:

```ts
export async function fetchListings(params: {
  category: CategoryId;
  page: number;
}): Promise<Page<Listing>> {
  const res = await fetch(`https://api.example.com/listings?category=${params.category}&page=${params.page}`);
  const data = await res.json();
  return {
    items: data.items,                                   // Listing[]
    nextPage: data.hasMore ? params.page + 1 : null,     // null = no more pages
  };
}
```

The hook, components, and screen need no changes — that's the whole point of
keeping the fetch behind this single function.
